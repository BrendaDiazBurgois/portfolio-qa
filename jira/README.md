# Jira – Gestión de Issues

En esta sección se documenta el uso de Jira para la gestión del proyecto QA.

## Tablero
Se configuró un tablero Kanban con el siguiente workflow:
- To Do
- In Development
- Ready for QA
- In QA
- Blocked
- Done

El objetivo es separar claramente las tareas de desarrollo y QA.

## User Stories
Se crearon historias de usuario con criterios de aceptación claros para permitir
el diseño de casos de prueba en Zephyr Scale.

## Bugs
Los defectos detectados durante la ejecución de pruebas fueron registrados en Jira
incluyendo pasos de reproducción, resultado esperado y resultado actual.
